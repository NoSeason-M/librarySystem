## 1. 项目概述

### 1.1 项目背景

当前大多数中小型图书馆（高校图书馆、院系资料室、社区图书室）仍依赖传统纸质登记或老旧的单机管理系统，存在效率低、数据孤岛、无法远程查询等问题。本项目旨在构建一套基于现代 Web 技术栈的图书管理系统，实现图书采编、流通、检索、统计的全流程数字化管理。

### 1.2 项目目标

- 实现图书基本信息、馆藏信息的标准化管理
- 支持读者自助查询、借阅、预约、续借等核心流通操作
- 为管理员提供采编、读者管理、违规处理、数据统计等后台能力
- 通过前后端分离架构，为后续扩展（移动端、小程序）预留接口

### 1.3 系统边界

| 包含 | 不包含 |
|---|---|
| 纸质图书的编目、流通、检索 | 电子书在线阅读 |
| 读者管理及借阅权限控制 | 自助借还硬件对接 |
| 逾期罚款计算与管理 | 第三方支付（罚款）对接 |
| 基础数据统计与可视化 | RFID/条码打印硬件驱动 |

## 2. 用户角色

系统划分为 **四种角色**，由低到高权限分级：

### 2.1 读者 (ROLE_READER)

- 在校学生/教职工/注册读者
- 可检索图书、查看个人借阅历史、借阅当前状态、预约图书、续借
- 不可进入后台管理界面

### 2.2 图书管理员 (ROLE_LIBRARIAN)

- 承担日常流通工作
- 可进行图书借出、归还、续借操作
- 管理读者账户（注册、挂失、解挂）
- 处理罚款
- 查看流通统计

### 2.3 采编员 (ROLE_CATALOGER)

- 专注于图书 metadata 管理
- 图书入库、编目、分类、馆藏地点管理
- 图书信息编辑与维护（封面、摘要、ISBN 校验）
- 批量导入/导出

### 2.4 系统管理员 (ROLE_ADMIN)

- 拥有全部权限
- 用户管理（创建/禁用/角色分配）
- 系统配置（借阅规则、罚款费率、开放时间）
- 操作日志审计
- 数据备份/恢复触发

## 4. 功能模块详细描述

### 4.1 图书搜索与发现

#### 4.1.1 普通搜索

- **入口**：首页顶部搜索框，全局可用
- **输入**：关键词（同时匹配书名、作者、ISBN、出版社）
- **交互**：支持拼音首字母模糊匹配（可选）
- **结果展示**：卡片列表 / 表格列表 两种视图切换
- **分页**：支持每页 10/20/50 条切换
- **空状态**：搜索无结果时显示「未找到相关图书，建议更换关键词」+ 推荐热门图书
- **加载态**：骨架屏占位

#### 4.1.2 高级搜索

- **入口**：搜索框旁的「高级搜索」按钮
- **表单字段**：
  - 书名（模糊匹配）
  - 作者（精确/模糊可切换）
  - ISBN（精确匹配）
  - 分类（树形下拉选择）
  - 出版社（下拉选择）
  - 出版年份范围（起始年 ~ 结束年）
  - 馆藏地（下拉选择）
  - 语言（中文/英文/其他）
  - 状态（全部/在馆/已借出）
- **排序**：相关度 / 出版时间倒序 / 借阅次数倒序 / 评分

#### 4.1.3 图书详情页

**布局**：左侧封面 + 右侧元信息 + 下方 Tabs

**元信息区域：**
- 书名、副标题、作者（可点击跳转该作者作品列表）
- ISBN、出版社、出版日期、页数、定价
- 分类路径（面包屑）
- 语言、装帧（平装/精装）
- 平均评分 + 评分人数
- 总馆藏量、可借数量、预约人数
- 二维码（ISBN 对应条码展示，可用于移动端扫码）

**Tabs 区域：**
1. **内容简介** — 摘要/目录
2. **馆藏信息** — 表格展示每本副本的条码号、馆藏地、当前状态（在馆/已借出/预约中/维修中）、应还日期
3. **借阅历史** — 近期的借阅记录（脱敏展示）
4. **相关推荐** — 基于相同分类或「借阅此书的人也借过」的推荐列表

**操作按钮：**
- 「预约」— 所有副本均已借出时可用
- 「收藏」— 加入个人书架
- 「分享」— 复制链接

#### 4.1.4 分类浏览

- 首页展示一级分类入口（图标 + 文字）
- 点击进入该分类下的图书列表，支持按二级/三级分类筛选
- 分类数据采用无限级树形结构（技术上支持三级即可）

#### 4.1.5 热门推荐

- **热门借阅**：按近 30 天借阅次数降序
- **新书上架**：按入库时间降序，带「NEW」标记
- **推荐图书**：管理员手动推荐（置顶标识）

### 4.2 借阅管理（核心流通）

#### 4.2.1 借书

- **入口**：管理员工作台 → 借书
- **操作流程**：
  1. 输入读者证号（或扫码）→ 自动加载读者信息及当前借阅数/限额
  2. 输入图书条码号（可连续扫码）→ 逐本添加
  3. 确认清单 → 提交 → 系统更新状态
- **校验规则**：
  - 读者是否存在且状态正常（未挂失/未禁用）
  - 读者当前借阅数是否达到上限
  - 图书状态是否为「在馆」
  - 是否有逾期未还图书（有则不允许再借）
  - 是否被他人预约（确认本预约者放弃后仍可借）
- **结果**：生成借阅记录，图书状态变更为「已借出」，应还日期 = 当前日期 + 借阅天数（按角色/读者类型配置）

#### 4.2.2 还书

- **入口**：管理员工作台 → 还书
- **操作流程**：
  1. 输入图书条码号 → 加载借阅记录
  2. 系统自动计算是否逾期
  3. 如有逾期，自动生成罚款记录（弹窗提示）
  4. 确认还书 → 更新图书状态为「在馆」
- **校验规则**：
  - 图书确实处于「已借出」状态
  - 如有预约队列，自动通知下一位预约者
  - 归还时检查图书是否损坏（管理员可标记损坏/丢失）

#### 4.2.3 续借

- **发起方**：读者（自助）/ 管理员（代操作）
- **条件**：
  - 距应还日期 ≤ 7 天
  - 未逾期（逾期图书不可续借，必须先还书）
  - 每本书最多续借 1 次（可配置）
  - 续借天数 = 原始借阅天数（可配置）
  - 该图书无其他读者预约
- **结果**：应还日期顺延

#### 4.2.4 预约

- **发起方**：读者
- **条件**：
  - 目标图书所有副本均「已借出」
  - 读者当前无同 ISBN 图书的在途预约
  - 读者未借阅该书的逾期副本
- **流程**：
  1. 读者点击「预约」
  2. 选择取书地点（如果有多馆藏地）
  3. 系统将读者加入预约队列（按时间排序）
  4. 当有副本被归还时，系统自动将预约状态变更为「待取书」
  5. 发送通知给预约者
  6. 预约保留时长（默认 2 天，可配置），超时未取则自动取消并顺延至下一位

#### 4.2.5 取书确认

- 读者到馆后，管理员扫描读者证 → 查看待取书列表 → 执行取书操作
- 系统将预约记录状态更新为「已完成」，同时生成借阅记录

#### 4.2.6 罚款管理

- **罚款类型**：
  - 逾期罚款：按天计算（费率可配置，可按读者类型区分）
  - 损坏赔偿：管理员手动录入金额
  - 丢失赔偿：管理员手动录入金额（通常为图书定价 × 倍数）
- **缴纳方式**：
  - 管理员收取（现金/扫码后标记已缴）
  - 标记为已缴后记录操作员
- **状态流转**：未缴 → 已缴 / 已豁免

### 4.3 图书管理（采编）

#### 4.3.1 图书编目

- **新增流程**：
  1. 填写 ISBN → 系统自动查询豆瓣/当当 API 预填元信息（可选）
  2. 手动补充/修正字段
  3. 上传封面图片
  4. 填写馆藏副本信息（条码号、馆藏地、价格等）
  5. 提交审核 → 状态为「待审核」或直接「已上架」（可配置）
- **必填字段**：书名、作者、ISBN、出版社、出版日期、分类、定价
- **选填字段**：副标题、译者、页数、装帧、摘要、目录、封面 URL、语种

#### 4.3.2 图书编辑

- 支持修改除 ISBN（作为唯一标识不可变更）外的所有字段
- 修改需记录操作日志
- 支持上下架操作（下架后读者端不可见）

#### 4.3.3 ISBN 查重

- 新增时自动检测 ISBN 是否已存在
- 如已存在，提示「该书已编目，请直接增加副本」

#### 4.3.4 批量导入

- 支持 Excel / CSV 模板导入
- 必含字段：书名、作者、ISBN
- 导入前校验数据格式
- 导入结果报告：成功条数、失败条数 + 失败原因逐行展示
- 支持分批导入（每次 ≤ 500 条）

#### 4.3.5 批量导出

- 支持按筛选条件导出图书列表为 Excel
- 导出的列可动态选择

#### 4.3.6 分类管理

- **无限级分类**（技术上支持三级）
- CRUD 操作
- 分类编码（行业标准：《中图法》分类号兼容）
- 每个分类可设定排序号

#### 4.3.7 出版社管理

- CRUD
- 包含名称、简称、联系电话、地址

#### 4.3.8 馆藏地点管理

- CRUD
- 字段：名称、编码、位置描述、是否启用
- 例：一楼社科书库、二楼科技书库、密集书库

#### 4.3.9 图书盘点

- 导出盘点清单（条码号列表）
- 扫码枪逐条扫描 → 系统比对 → 生成盘亏/盘盈报告
- 异动列表：应有未扫到（盘亏） / 应无但扫到（盘盈）

### 4.4 读者管理

#### 4.4.1 读者注册

- **管理员代注册**：
  - 填写姓名、证件号、读者类型（学生/教师/校外）、联系电话、邮箱
  - 系统生成读者证号（规则可配置：如 8 位自增数字或年份前缀）
  - 初始密码（身份证后 6 位或默认密码）
- **读者自助注册**（可选）：
  - 在线填写信息 → 管理员审核通过后激活

#### 4.4.2 读者信息管理

- 编辑基本信息
- 挂失/解挂
- 冻结/解冻
- 重置密码

#### 4.4.3 读者类型配置

- 类型名称（学生/教师/校外读者等）
- 最大借阅数量
- 借阅天数
- 续借次数限制
- 逾期费率（元/天）
- 预约最大保留天数

#### 4.4.4 读者借阅历史

- 当前借阅：显示书名、借出日期、应还日期、剩余天数
- 历史记录：已归还的记录，支持按时间范围筛选
- 预约记录：当前预约及历史预约

### 4.7 统计分析

#### 4.7.1 借阅统计

- **时间维度**：按日/周/月/年聚合
- **图表类型**：
  - 折线图：借阅量趋势
  - 柱状图：对比不同分类借阅量
  - 饼图：读者类型借阅占比
- **筛选器**：时间范围、分类、读者类型

#### 4.7.2 热门图书排行

- 借阅次数 Top N（支持 N 配置，默认 20）
- 可按日/周/月/年/全部统计
- 表格 + 柱状图展示

#### 4.7.3 流通统计

- 当日借书/还书数量
- 逾期率 = 逾期记录 / 总借阅记录
- 预约兑现率 = 已取书预约 / 总预约

#### 4.7.4 馆藏统计

- 总藏书量、总副本数
- 各分类藏书占比
- 各馆藏地分布
- 月度新增图书趋势

#### 4.7.5 读者统计

- 总注册读者数
- 活跃读者数（近 30 天有借阅记录）
- 读者类型分布

### 4.8 系统管理

#### 4.8.1 用户管理

- 用户列表（搜索、分页）
- 创建用户（用户名、姓名、邮箱、角色）
- 编辑用户
- 启用/禁用用户
- 删除用户（逻辑删除或限制有借阅记录的用户不可删除）

#### 4.8.2 角色管理

- 预置四种角色，支持自定义扩展
- 每个角色的权限以树形展示（勾选）
- 权限颗粒度：菜单权限 + 按钮权限（如「新增图书」是一个独立权限点）

#### 4.8.3 操作日志

- 记录用户关键操作：增删改图书、借还书、用户管理等
- 字段：操作人、操作时间、操作类型、IP 地址、操作详情
- 支持按时间范围/操作人/操作类型筛选
- 日志不可删除（append-only，可设置自动归档策略）

#### 4.8.4 系统参数配置

- 借阅规则：最大借阅数、借阅天数、续借次数、续借天数
- 罚款规则：逾期费率、损坏费率倍数、丢失费率倍数
- 预约规则：预约保留时长（小时）
- 借阅开放时间：每日开放时段（仅做显示参考）
- 读者初始密码
- 界面配置：系统名称、Logo、备案号、联系邮箱

#### 4.8.5 数据备份

- 手动触发备份
- 备份列表（文件名、大小、创建时间）
- 下载备份文件
- 恢复备份（风险提示 + 二次确认）

#### 4.8.6 公告管理

- CRUD
- 支持 Markdown 富文本
- 置顶功能
- 发布范围：全部 / 仅读者 / 仅管理员
- 定时发布（可选）

#### 4.8.7 数据字典

- 分类字典：图书分类、读者类型、罚款类型等
- 支持前端直接维护字典项
- 字典编码作为业务字段关联

## 5. 非功能性需求

### 5.1 性能要求

| 指标 | 目标值 |
|---|---|
| 页面首次加载 | ≤ 2.5s（首屏） |
| 接口响应时间（读） | ≤ 200ms（95 分位） |
| 接口响应时间（写） | ≤ 500ms（95 分位） |
| 搜索响应时间 | ≤ 1s |
| 并发支持 | ≥ 100 同时在线 |
| 数据库单表规模 | ≥ 10 万级不出现明显性能下降 |

### 5.2 安全要求

- 密码使用 BCrypt 加密存储
- 登录验证码（图形验证码，连续 3 次错误后必输）
- JWT Token 认证，Access Token 有效期 2h，Refresh Token 有效期 7d
- API 防重放（可选）、XSS 过滤、SQL 注入防护（MyBatis-Plus 参数化查询）
- 敏感操作（借书、还书、罚款）记录操作日志
- 前后端参数双重校验
- 接口权限校验（Spring Security + 自定义权限注解）

### 5.3 可用性要求

- 系统 7×24h 运行
- 关键操作（借还书）的失败率 < 0.1%
- 核心业务流程不超过 5 步完成

### 5.4 可维护性要求

- 前端统一组件封装（Element Plus 二次封装）
- 后端按模块分包（controller / service / mapper / entity / dto）
- API 遵循 RESTful 规范
- 统一异常处理 + 统一响应体格式
- 提供 Swagger / Knife4j 在线接口文档

### 5.5 兼容性要求

- **浏览器**：Chrome ≥ 90, Edge ≥ 90, Firefox ≥ 90, Safari ≥ 14
- **分辨率**：适配 1920×1080 为主，兼容 1366×768 ~ 2560×1440
- **移动端**：读者端页面响应式布局（TailwindCSS 栅格系统）
